The `top` command displays a real-time view of the running processes on the system and their resource utilization. Here's a breakdown of the columns:

- `RES`: The amount of physical memory (RAM) used by the process.
- `PID`: The process ID.  
- `USER`: The user running the process.  
- `PRI`: The priority of the process (higher value means lower priority).  
- `NI`: The nice value, which influences the scheduled priority.  
- `VIRT`: The total amount of virtual memory used by the process.  
- `SHR`: The amount of shared memory used by the process.  
- `S`: The process status (R=Running, S=Sleeping, etc).  
- `%CPU`: The percentage of CPU time used by the process.  
- `%MEM`: The percentage of physical RAM used by the process.  
- `TIME+`: The total CPU time used by the process.
- `Command`: The command that launched the process.

This allows you to identify processes consuming excessive resources. You can sort the list by various columns, view additional info, change delay intervals, and more.
